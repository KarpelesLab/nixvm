    Static binaries must be statically-linked ELF64, little-endian, aarch64
    (EM_AARCH64) or x86-64 (EM_X86_64) executables —
    nixvm auto-detects which from the ELF header and runs it on the matching
    software interpreter (see src/vcpu/interp.rs /
    src/vcpu/interp_x86.rs). There is no guest filesystem here
    beyond an in-memory /, /tmp, /dev,
    /proc, /sys, so binaries that only touch
    stdout/stderr/exit work best. A syscall nixvm doesn't implement yet
    returns -ENOSYS to the guest and is tallied instead of
    crashing the run — see the stats line above.
